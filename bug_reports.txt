Баг-репорт содержит ответы на следующие вопросы:

что идёт не так;
где проявляется дефект;
когда ошибка воспроизводится.

Форма заполнения баг-репорта включает несколько полей (атрибутов).
1.Summary (заголовок)
Проект (Project)
Компонент приложения (Component)
Номер версии (Version)	
Серьезность (Severity)
Приоритет (Priority)
Статус (Status)	
Автор (Author)	
Назначен на (Assigned To)

Окружение
ОС / Сервис Пак и т.д. / Браузера + версия / ....

Описание
Шаги воспроизведения (Steps to Reproduce)
Фактический Результат (Result)	
Ожидаемый результат (Expected Result)

Дополнения
Прикрепленный файл (Attachment)
